如何取得設定

步驟 1：建立 Firebase 專案

  1. 前往 Firebase 控制台。
  2. 點擊「建立專案」，並為您的專案命名。

步驟 2：建立 Firestore 資料庫

  1. 在左側選單點擊「建構」 > 「Firestore Database」。
  2. 點擊「建立資料庫」，選擇「以測試模式啟動」。
  3. 選擇位置後啟用。

步驟 3：取得設定碼

  1. 點擊「專案總覽」旁的齒輪 ⚙️ > 「專案設定」。
  2. 在下方「您的應用程式」區塊，點擊 </> (Web) 圖示。
  3. 註冊暱稱後，您會看到一段程式碼。
  4. 複製 `const firebaseConfig = { ... };` 大括號中間的 6 行內容。
  5. 回到本系統，點擊右上角的設定 (⚙️) 按鈕，貼上並儲存。

步驟 4：設定權限 (重要！)

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

請將上述規則貼到 Firestore 的「規則」分頁中並發佈。

附錄：常見錯誤排除

問題：點擊「查看紀錄」時沒有反應或跳出錯誤。

原因：這是第一次查詢歷史紀錄時會發生的正常情況。Firebase 需要為「學生ID」和「時間戳」的組合查詢建立一個索引（就像書本的目錄），才能快速地排序與尋找資料。

解決方法：

  1. 在瀏覽器中按下 F12 鍵打開「開發人員工具」，並切換到「Console (主控台)」分頁。
  2. 您應該會看到一條來自 Firebase 的錯誤訊息，內容包含 "The query requires an index"。
  3. 這條錯誤訊息中會包含一個可點擊的連結，看起來像這樣：https://console.firebase.google.com/v1/r/project/...
  4. 點擊那個連結。它會直接帶您前往 Firebase 網站的索引建立頁面，並且所有需要的設定都已自動填好。
  5. 在 Firebase 頁面中，直接點擊「建立索引」按鈕。
  6. 索引建立需要幾分鐘的時間。您可以在 Firebase 網站上看到建立進度。
  7. 索引建立完成後，回到本頁面重新整理，「查看紀錄」功能現在應該就可以正常運作了。
